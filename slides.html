The Challenge

LLMs Generate Code...

But we can't verify if it's correct without testing

  • Code looks syntactically correct
  • May contain subtle semantic errors
  • Testing is time-consuming

The Black Box Problem

Neural networks are notoriously opaque

Polysemanticity

Individual neurons respond to multiple, unrelated concepts—making interpretation nearly impossible

2

Our Approach: Sparse Autoencoders

We use SAEs to decompose neural activations into interpretable features

3

Key Discovery: The Asymmetry

We found code correctness directions exist—but they work asymmetrically:

F1 0.821
Incorrect Predicting
4.04%
Correct Steering
Failed
Incorrect Steering

The Insight

Models encode "wrongness" as detectable anomalies, but "correctness" is more fundamental and harder to manipulate

4

Practical Applications

  • Error Detection Systems: Use incorrect-predicting directions (F1=0.821) to flag likely buggy code before deployment
  • Prompting Best Practices: Prioritize test examples over problem descriptions—test cases drive attention +15%
  • Selective Steering: Two-stage approach: predict errors first, then steer only flagged samples to avoid corrupting correct code

No Retraining Required

These techniques work with existing models through simple prompt restructuring and activation analysis
